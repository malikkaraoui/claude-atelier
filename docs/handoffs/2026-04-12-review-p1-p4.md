## Réponse de : Copilot/GPT (GPT-5.4, 2026-04-12)

### Verdict global

Le projet est plus solide que la moyenne pour un package de config agentique.
6 fragilités sérieuses identifiées.

### 7 points classés par gravité

1. **CLI publié cassera** — `test/` n'est pas dans `files` de `package.json`.
   Donc `claude-atelier lint` et `claude-atelier doctor` crasheront après `npm publish`.
2. **`init --lang en` expose un mode non supporté** — installe partiellement sans CLAUDE.md.
3. **`init` écrase silencieusement** — `copyFileSync` sur `.claude/` et `scripts/` sans
   demander. Risque d'écraser des configs locales customisées sur un vrai projet.
4. **`doctor` = presence checker, pas smoke test** — ne vérifie ni tarball, ni install
   globale, ni hooks câblés, ni cohérence .claudeignore/.gitignore.
5. **README/roadmap/CI publique pas alignés** — README dit encore que init/doctor sont
   des stubs. CI publique montrait un run failed.
6. **hooks annoncés mais non livrés** — `hooks/` est vide, `settings.json` n'a pas de
   bloc hooks, `doctor` ne contrôle rien de ça.
7. **`rules/` vide + `CLAUDE.project.md` absent** — promesses structurelles non tenues.

### Checks doctor manquants (par priorité)

- Très prioritaires : tarball publishable, smoke init, mode EN dégradé
- Prioritaires : hooks livrés + câblés, template completeness, ignore coherence
- Utiles : version manifest, drift détection, idempotence init

### Satellites fragiles (périssabilité)

`subagents.md`, `hooks.md`, `plugins.md`, `qmd-integration.md` — décrivent des
surfaces mouvantes (noms de subagents, événements hooks, APIs tierces).
Frontmatter encore en état intermédiaire (`pas encore reference, a ajouter en P3.e`).
